## 1\. Assignment 엔티티 — `assignment/domain/Assignment.java`

연관관계의 **주인(N쪽)**. 외래 키 `member_id`를 관리한다.

```java
package com.lielion.PBL.assignment.domain;

import com.lielion.PBL.member.domain.Member;
import jakarta.persistence.*;

@Entity
public class Assignment {

    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    private String title;         // 과제 제목
    private String description;    // 과제 설명

    // N:1 — 여러 Assignment가 하나의 Member에 속한다. 이 엔티티가 연관관계의 주인.
    // @JoinColumn: assignment 테이블에 member_id 외래 키 컬럼을 만든다.
    @ManyToOne
    @JoinColumn(name = "member_id")
    private Member member;

    // JPA가 프록시/리플렉션용으로 요구하는 기본 생성자 (외부 직접 호출 방지를 위해 protected)
    protected Assignment() {
    }

    // 생성용 생성자
    public Assignment(String title, String description, Member member) {
        this.title = title;
        this.description = description;
        this.member = member;
    }

    // 수정용 메서드 — 영속 상태에서 이 메서드로 값을 바꾼다.
    public void updateInfo(String title, String description) {
        this.title = title;
        this.description = description;
    }

    // Getter
    public Long getId() {
        return id;
    }

    public String getTitle() {
        return title;
    }

    public String getDescription() {
        return description;
    }

    public Member getMember() {
        return member;
    }
}
```

package com.lielion.PBL.assignment.dto;

import com.lielion.PBL.assignment.domain.Assignment;

public class AssignmentResponse {

    private Long id;
    private String title;
    private String description;
    private Long memberId;
    private String memberName;

    public AssignmentResponse(Long id, String title, String description,
                              Long memberId, String memberName) {
        this.id = id;
        this.title = title;
        this.description = description;
        this.memberId = memberId;
        this.memberName = memberName;
    }

    public static AssignmentResponse from(Assignment assignment) {
        return new AssignmentResponse(
                assignment.getId(),
                assignment.getTitle(),
                assignment.getDescription(),
                assignment.getMember().getId(),
                assignment.getMember().getName()   // week8 Member의 getName() 사용
        );
    }

    public Long getId() {
        return id;
    }

    public String getTitle() {
        return title;
    }

    public String getDescription() {
        return description;
    }

    public Long getMemberId() {
        return memberId;
    }

    public String getMemberName() {
        return memberName;
    }
}

## 5\. Service — `assignment/service/AssignmentService.java`

클래스는 `@Transactional(readOnly = true)`, **변경 메서드(등록·수정·삭제)에만** `@Transactional`을 개별로 붙여 읽기 전용을 해제한다. 에러 처리는 **null 반환**.

```java
package com.lielion.PBL.assignment.service;

import com.lielion.PBL.assignment.domain.Assignment;
import com.lielion.PBL.assignment.repository.AssignmentRepository;
import com.lielion.PBL.member.domain.Member;
import com.lielion.PBL.member.repository.MemberRepository;
import org.springframework.stereotype.Service;
import org.springframework.transaction.annotation.Transactional;

import java.util.List;

@Service
@Transactional(readOnly = true) // 기본은 읽기 전용 → 변경 감지 생략으로 조회 성능 이점
public class AssignmentService {

    private final AssignmentRepository assignmentRepository;
    private final MemberRepository memberRepository;

    public AssignmentService(AssignmentRepository assignmentRepository,
                             MemberRepository memberRepository) {
        this.assignmentRepository = assignmentRepository;
        this.memberRepository = memberRepository;
    }

    // 과제 등록 — 데이터 변경이므로 @Transactional로 읽기 전용 해제
    @Transactional
    public Assignment create(Long memberId, String title, String description) {
        Member member = memberRepository.findById(memberId).orElse(null);
        if (member == null) {
            return null; // 멤버 없음 → null 반환 (Controller가 404 처리)
        }
        Assignment assignment = new Assignment(title, description, member);
        return assignmentRepository.save(assignment);
    }

    // 멤버별 과제 조회 — 클래스 레벨 readOnly = true 적용
    public List<Assignment> findByMemberId(Long memberId) {
        return assignmentRepository.findByMemberId(memberId);
    }

    // 단건 조회 — 없으면 null
    public Assignment findById(Long id) {
        return assignmentRepository.findById(id).orElse(null);
    }

    // 수정 — 데이터 변경이므로 @Transactional
    @Transactional
    public Assignment update(Long id, String title, String description) {
        Assignment assignment = assignmentRepository.findById(id).orElse(null);
        if (assignment == null) {
            return null;
        }
        assignment.updateInfo(title, description);
        return assignmentRepository.save(assignment);
    }

    // 삭제 — 데이터 변경이므로 @Transactional
    @Transactional
    public void delete(Long id) {
        assignmentRepository.deleteById(id);
    }
}
```

## 6\. Controller — `assignment/controller/AssignmentController.java`

null 반환을 받아 상태코드로 변환한다. (등록 201 / 조회 200 / 수정 200 / 삭제 204 / 없음 404)

```java
package com.lielion.PBL.assignment.controller;

import com.lielion.PBL.assignment.domain.Assignment;
import com.lielion.PBL.assignment.dto.AssignmentCreateRequest;
import com.lielion.PBL.assignment.dto.AssignmentResponse;
import com.lielion.PBL.assignment.dto.AssignmentUpdateRequest;
import com.lielion.PBL.assignment.service.AssignmentService;
import org.springframework.http.HttpStatus;
import org.springframework.http.ResponseEntity;
import org.springframework.web.bind.annotation.*;

import java.util.List;
import java.util.stream.Collectors;

@RestController
public class AssignmentController {

    private final AssignmentService assignmentService;

    public AssignmentController(AssignmentService assignmentService) {
        this.assignmentService = assignmentService;
    }

    // 과제 등록: POST /members/{memberId}/assignments
    @PostMapping("/members/{memberId}/assignments")
    public ResponseEntity<AssignmentResponse> create(
            @PathVariable Long memberId,
            @RequestBody AssignmentCreateRequest request) {

        Assignment assignment =
                assignmentService.create(memberId, request.getTitle(), request.getDescription());

        if (assignment == null) {
            return ResponseEntity.status(HttpStatus.NOT_FOUND).build(); // 404: 멤버 없음
        }
        return ResponseEntity.status(HttpStatus.CREATED) // 201
                .body(AssignmentResponse.from(assignment));
    }

    // 멤버별 과제 목록: GET /members/{memberId}/assignments
    @GetMapping("/members/{memberId}/assignments")
    public ResponseEntity<List<AssignmentResponse>> findByMember(@PathVariable Long memberId) {
        List<AssignmentResponse> responses = assignmentService.findByMemberId(memberId).stream()
                .map(AssignmentResponse::from)
                .collect(Collectors.toList());
        return ResponseEntity.ok(responses); // 200
    }

    // 과제 단건 조회: GET /assignments/{id}
    @GetMapping("/assignments/{id}")
    public ResponseEntity<AssignmentResponse> findOne(@PathVariable Long id) {
        Assignment assignment = assignmentService.findById(id);
        if (assignment == null) {
            return ResponseEntity.status(HttpStatus.NOT_FOUND).build(); // 404
        }
        return ResponseEntity.ok(AssignmentResponse.from(assignment)); // 200
    }

    // 과제 수정: PUT /assignments/{id}
    @PutMapping("/assignments/{id}")
    public ResponseEntity<AssignmentResponse> update(
            @PathVariable Long id,
            @RequestBody AssignmentUpdateRequest request) {

        Assignment assignment =
                assignmentService.update(id, request.getTitle(), request.getDescription());

        if (assignment == null) {
            return ResponseEntity.status(HttpStatus.NOT_FOUND).build(); // 404
        }
        return ResponseEntity.ok(AssignmentResponse.from(assignment)); // 200
    }

    // 과제 삭제: DELETE /assignments/{id}
    @DeleteMapping("/assignments/{id}")
    public ResponseEntity<Void> delete(@PathVariable Long id) {
        // 없는 id면 404를 주려면 존재 확인 후 삭제
        if (assignmentService.findById(id) == null) {
            return ResponseEntity.status(HttpStatus.NOT_FOUND).build(); // 404
        }
        assignmentService.delete(id);
        return ResponseEntity.noContent().build(); // 204
    }
}
```

# 📝 과제 목표 질문 — 답변

> "~설명할 수 있다" 학습 목표 7개에 대한 답변입니다. 제출/구술 그대로 활용하세요.

### Q1. `@ManyToOne`과 `@OneToMany`의 역할과 차이는?

`@ManyToOne`은 \*\*N쪽에 붙어 "나는 여러 개 중 하나이고, 상대 하나를 참조한다"\*\*를 뜻한다. 여기선 여러 Assignment가 한 Member를 가리키므로 `Assignment.member`에 붙는다. `@OneToMany`는 \*\*1쪽에 붙어 "나는 하나이고, 여러 개를 가진다"\*\*를 뜻하며 `Member.assignments`(컬렉션)에 붙는다. 핵심 차이는 **실제 외래 키를 관리하는 쪽은** `@ManyToOne`**(N쪽)뿐**이고, `@OneToMany`는 `mappedBy`로 그 관계를 되비추는 조회용이라는 점이다.

### Q2. 연관관계의 주인(owning side)은 무엇이고, 왜 "N"쪽이 주인이 되나?

연관관계의 주인은 **외래 키를 실제로 관리(등록·수정)하는 쪽**이다. DB에서 외래 키 `member_id`는 항상 **N쪽인** `assignment` **테이블**에 생긴다. 관계를 바꾼다는 건 결국 이 `member_id` 값을 바꾸는 일이므로, 그 컬럼을 가진 **Assignment(N쪽)가 주인**이 되는 것이 자연스럽고 SQL과 1:1로 대응된다. 만약 1쪽(Member)을 주인으로 두면 객체를 바꿨는데 엉뚱한 테이블로 UPDATE가 나가 혼란스럽고 추가 쿼리도 발생한다.

### Q3. `mappedBy`는 무엇을 의미하나?

`@OneToMany(mappedBy = "member")`는 \*\*"이 관계의 외래 키는 내가 아니라 상대(Assignment)의 `member` 필드가 관리한다"\*\*는 선언이다. 즉 Member쪽 컬렉션은 \*\*주인이 아니라 읽기 전용(조회용)\*\*이라는 표시다. 그래서 `member` 테이블에는 Assignment 관련 컬럼이 생기지 않고, 컬렉션에 값을 넣고 빼는 것만으로는 DB 외래 키가 바뀌지 않는다.

### Q4. `@JoinColumn`의 역할은?

`@JoinColumn(name = "member_id")`은 **이 연관관계를 DB에서 어떤 외래 키 컬럼으로 표현할지 지정**한다. 이 설정 때문에 `assignment` 테이블에 `member_id`라는 외래 키 컬럼이 만들어지고, `member` 테이블의 PK를 참조하게 된다. 이름을 지정하지 않으면 JPA가 기본 규칙(`member_id` 형태)으로 자동 생성하지만, 명시하면 컬럼명을 명확히 통제할 수 있다.

### Q5. DB의 외래 키(Foreign Key)는 어떤 테이블에 생기나?

**N쪽 테이블, 즉** `assignment` **테이블**에 `member_id` 외래 키가 생긴다. 1:N 관계에서 외래 키는 언제나 "여러 개"인 쪽에 둬야 각 행이 자신이 속한 부모 하나를 가리킬 수 있기 때문이다. 반대로 하면(1쪽에 두면) 한 회원이 가진 여러 과제를 한 컬럼에 담을 수 없다.

### Q6. `@Transactional`이 무엇이고, 서비스 계층에 붙이는 이유는?

`@Transactional`은 **여러 DB 작업을 하나의 단위(트랜잭션)로 묶어, 전부 성공하면 커밋하고 중간에 예외가 나면 전체를 롤백**해주는 기능이다. 서비스 계층에 붙이는 이유는, **하나의 비즈니스 작업(예: 과제 등록 = 멤버 조회 + 저장)이 여러 DB 호출로 이뤄지기 때문**이다. 이 묶음이 서비스 메서드 단위로 원자적으로 처리돼야 중간 실패 시 데이터가 반쯤 저장되는 사고를 막을 수 있다. Repository 단위로 걸면 앞 작업이 이미 커밋돼 되돌릴 수 없다.

### Q7. `@Transactional(readOnly = true)`를 조회 메서드에 쓰는 이유는?

읽기 전용 트랜잭션에서는 **JPA가 변경 감지(dirty checking)를 위한 스냅샷 비교와 flush를 생략**한다. 조회만 하는 메서드는 어차피 UPDATE가 필요 없으므로, 이 과정을 건너뛰어 **불필요한 오버헤드를 줄이고 성능 이점**을 얻는다. 그래서 클래스 레벨에 `readOnly = true`를 기본으로 깔고, 데이터를 바꾸는 등록·수정·삭제 메서드에만 `@Transactional`을 따로 붙여 읽기 전용을 해제하는 전략을 쓴다.

### 제출 전 마지막 점검

*   패키지가 `com.lielion.PBL.assignment...` 구조인가
    
*   Assignment 필드가 `id / title / description / member`인가 (content·status 없음)
    
*   에러 처리가 **null 반환 + Controller null 체크**인가 (커스텀 예외 미사용)
    
*   `@Transactional(readOnly=true)`는 클래스, 변경 메서드엔 `@Transactional` 개별 적용했는가
    
*   `mappedBy = "member"`는 Member(1쪽)에, `@JoinColumn`은 Assignment(N쪽)에 있는가
    
*   실행 시 `assignment` 테이블에 `member_id` 외래 키가 생성됐는가
