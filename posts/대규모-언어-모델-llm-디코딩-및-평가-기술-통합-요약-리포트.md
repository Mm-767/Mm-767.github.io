## Chapter 1. 대규모 언어 모델(LLM) 디코딩 기술 (Decoding of LLMs)

## 1.1 언어 모델의 정의 및 자기회귀 생성 메커니즘

### Core Concept

*   \*\*언어 모델(Language Model)\*\*의 본질은 주어진 단어 또는 토큰 시퀀스에 대해 통계적인 확률 분포를 할당하는 수학적 모델입니다 \[37, 38\].
    
*   현대 디코더 전용(Decoder-only) LLM은 **자기주도 학습(Self-Supervised Learning)** 패러다임을 따릅니다 \[37, 38\]. 별도의 사람이 레이블링한 데이터 없이, 방대한 인터넷 코퍼스 내에서 다음에 나타날 토큰을 예측하는 **Next Token Prediction** 태스크를 수행하며 사전 학습됩니다 \[37, 38, 59\].
    
*   추론 단계에서는 이전 시점까지 예측 및 생성된 모든 토큰들을 고정 입력 컨텍스트(Input Context)로 환류(Autoregressive Loop)하여 다음 토큰을 하나씩 순차적으로 생성하는 **인과적 자기회귀 생성(Causal Autoregressive Generation)** 방식으로 작동합니다 \[37, 38, 59\].
    
*   토큰 생성이 진행되다가 사전 학습 과정에서 정의된 문장 종결 토큰인 **EOS(End-of-Sentence) 토큰**이 기출되면 순차적 추론 루프가 최종적으로 종료됩니다 \[37\].
    

### Mathematical Formulation (If any)

*   **자기회귀 모델의 체인 룰(Chain Rule)에 기반한 조건부 누적 확률 공식**: $$p(x\_{1:L}) = p(x\_1, x\_2, \\dots, x\_L) = \\prod\_{i=1}^{L} p(x\_i \\mid x\_{<i})$$
    
    *   $x\_{1:L}$: 전체 생성된 길이 $L$의 토큰 시퀀스입니다 \[37, 38\].
        
    *   $x\_i$: $i$번째 생성된 토큰입니다 \[37\].
        
    *   $x\_{<i}$: $i$번째 토큰 생성 이전에 조건부 문맥으로 작용하는 전반부 시퀀스 $\[x\_1, \\dots, x\_{i-1}\]$입니다 \[16, 37\].
        
    *   $p(x\_i \\mid x\_{<i})$: 고정된 가중치 파라미터 $\\theta$를 갖는 트랜스포머 네트워크의 최종 투영층(Projection Layer)이 뿜어내는 로짓(Logit)에 Softmax 연산을 적용하여 얻은, 어휘 사전 $\\mathcal{V}$ 전체에 대한 이산 확률 분포입니다 \[37, 38\].
        

### Key Tradeoffs / Procedures

*   **인과적 마스킹(Causal Masking)의 절차**:
    
    1.  트랜스포머 디코더 블록 내부의 자가 주의집중(Self-Attention) 레이어에서 입력 임베딩 행렬 $X$를 각각 $Q$ (Query), $K$ (Key), $V$ (Value) 행렬로 선형 투영합니다 \[37, 38\].
        
    2.  쿼리 $Q\_i$와 키 $K\_j^\\top$의 내적 계산으로 원시 어텐션 점수 행렬 $S\_{ij} = \\frac{Q\_i K\_j^\\top}{\\sqrt{d}}$를 생성합니다 \[100, 37, 38\].
        
    3.  미래 시점의 정보 누출(Data Leakage)을 원천 차단하기 위해 하삼각(Lower-triangular) 형태의 인과적 마스크 $M \\in \\mathbb{R}^{n \\times n}$를 정의하여 덧셈 연산을 적용합니다 \[100, 37, 38\]. $$M\_{ij} = \\begin{cases} 0 & \\text{if } j \\le i \\ -\\infty & \\text{if } j > i \\end{cases}$$
        
    4.  Softmax 분모 연산에서 $e^{-\\infty} = 0$이 되므로, 임의의 시점 $i$의 토큰은 오직 $j \\le i$ 범위의 과거 맥락에만 주의집중하게 됩니다 \[100, 38, 59\].
        
    
    *   _주의 사항_: 만약 마스킹 점수에 $-\\infty$ 대신 단순히 물리적 0을 적용하면 Softmax 계산 시 $e^0 = 1$이 되어 미래의 정보가 누수되는 수치적 오류가 발생하므로 반드시 극단적인 음의 무한대 마스킹을 유지해야 합니다 \[59\].
        

## 1.2 토크나이제이션(Tokenization) 및 서브워드 알고리즘 분석

### Core Concept

*   \*\*토크나이제이션(Tokenization)\*\*이란 텍스트 문자열을 트랜스포머 모델이 임베딩 벡터로 처리할 수 있도록 물리적으로 독립된 이산적 단위인 토큰(Token)으로 분할하고, 이를 고유 정수 ID에 매핑하는 전처리 과정입니다 \[37, 38\].
    
*   문자 수준(Character-level) 방식은 사전(Vocabulary) 크기를 최소화하여 메모리 소모를 극적으로 줄일 수 있으나 문맥의 시퀀스 길이가 너무 늘어나 계산 효율이 급감합니다 \[37, 38\]. 반면 단어 수준(Word-level) 방식은 미등록 단어(OOV, Out-of-Vocabulary) 처리가 불가능해지는 한계가 있습니다 \[37, 38\].
    
*   현대 LLM은 이 두 한계를 극복하기 위해 단어를 더 유연한 서브 형태소 단위로 쪼개는 \*\*서브워드 토큰화(Subword Tokenization)\*\*를 표준으로 정착시켰습니다 \[37, 38\].
    

### Mathematical Formulation (If any)

*   **BPE (Byte-Pair Encoding) 병합 메커니즘**: 상향식(Bottom-up)으로 동작하며, 텍스트 데이터 내부에서 가장 빈도가 높은 인접 문자 쌍(Pair)을 결합 테이블에 점진적으로 추가합니다 \[37, 38\]. 이때 단어 경계 마커 `</w>` 또는 SentencePiece 특수 공백 기호 `_`를 삽입하여 단어 구조를 보존합니다 \[37, 38\].
    
*   **WordPiece 우도 극대화 공식**: BPE처럼 단순히 빈도가 가장 높은 쌍을 합치지 않고, 두 토큰 $A$와 $B$를 병합했을 때의 언어 모델 우도(Likelihood) 증가량이 가장 높은 쌍을 탐욕적으로 선택합니다 \[37, 38\]. $$\\text{Score}(A, B) = \\frac{\\text{count}(A, B)}{\\text{count}(A) \\times \\text{count}(B)}$$
    
    *   이 공식은 분모에 개별 토큰의 단독 빈도수를 배치함으로써, 각 토큰이 결합하지 않고 따로 쓰이는 확률에 비해 두 토큰이 함께 인접하여 기출될 상호 의존도가 얼마나 높은지 통계적으로 평가합니다 \[37, 38\].
        
*   **Unigram 서브워드 정규화 확률 모델**: 하향식(Top-down) 기법으로, 임의의 대형 사전 후보 세트에서 시작하여 말뭉치의 손실을 최소화하는 방향으로 사전 크기를 축소시킵니다 \[37, 38\]. 단어 $x$의 최적 분할 $S(x)$는 단어 사전에 수록된 하위 세그먼트들의 확률곱을 최대화하는 경로로 구성됩니다 \[37\]. $$P(S(x)) = \\prod\_{s \\in S(x)} p(s), \\quad \\text{subject to } \\sum\_{s \\in \\mathcal{V}} p(s) = 1$$
    

### Key Tradeoffs / Procedures

*   **서브워드 알고리즘별 특징 비교 및 절차**:
    
    *   **BPE (Byte-Pair Encoding)**:
        
        *   _절차_: 모든 문자를 개별 토큰으로 초기화 $\\rightarrow$ 말뭉치 내 인접 빈도가 최대인 쌍 병합 $\\rightarrow$ 지정된 사전 크기 도달 시 종료 \[37, 38\].
            
        *   _장단점_: 영어 및 알파벳 계열의 도메인에서 압축율이 높고 강건하게 대처 가능하나, 문자 경계를 인위적으로 구분하기 위한 복잡한 사전 처리 장치가 수반되어야 합니다 \[37, 38\].
            
    *   **SentencePiece**:
        
        *   _절차_: 공백 문자 자체를 특수 문자(예: `_`)로 명시 치환 $\\rightarrow$ 사전 전처리 분할(Space Tokenization) 없이 원시 바이트 스트림에 다이렉트 적용 \[37, 38\].
            
        *   _장단점_: 아시아권 언어(한국어, 중국어, 일본어 등)처럼 띄어쓰기가 복잡하거나 형태소 결합이 다양한 언어에 최적의 효율성을 보여줍니다 \[37, 38\].
            

## 1.3 결정론적 및 샘플링 제어 디코딩 기법

### Core Concept

*   트랜스포머의 최종 출력 벡터는 사전에 정의된 차원 $|\\mathcal{V}|$를 지닌 원시 점수(Raw logits) 형태를 취하므로, 이를 디코딩 전략을 통해 텍스트로 전환해야 합니다 \[37, 38\].
    
*   \*\*결정론적 디코딩(Deterministic Decoding)\*\*은 높은 일관성을 보장하지만 문장 순환 반복(Degeneration)에 취약합니다 \[37, 38\]. \*\*확률적 샘플링 디코딩(Stochastic Sampling)\*\*은 무작위 다양성을 주입하지만 문맥 탈선과 의미적 붕괴가 발생할 수 있습니다 \[37, 38\].
    

### Mathematical Formulation (If any)

*   **그리디 디코딩(Greedy Decoding)**: $$x\_t = \\arg\\max\_{w \\in \\mathcal{V}} p(w \\mid x\_{<t})$$
    
*   **빔 서치(Beam Search)의 장기 로그 우도 최대화**: $$\\text{Score}(x\_{1:t}) = \\sum\_{\\tau=1}^{t} \\log p(x\_\\tau \\mid x\_{1:\\tau-1})$$
    
    *   매 시점 $t$마다 고정된 빔 폭(Beam Width) $k$개의 최적 누적 확률 시퀀스만 유지하여 전개합니다 \[37, 38\].
        
*   **온도 제어(Temperature Control)를 적용한 Softmax**: $$p(x\_i \\mid x\_{<i}) = \\frac{\\exp(z\_i / T)}{\\sum\_{j} \\exp(z\_j / T)}$$
    
    *   $z\_i$: 사전 매핑 전 단어 $i$의 원시 로짓 점수입니다 \[37, 38\].
        
    *   $T$: 생성 가변성을 조절하는 하이퍼파라미터입니다 \[37, 38\].
        
        *   $T > 1$ 일 때: 분포가 평탄화(Smoothing)되어 극단적으로 낮은 확률의 토큰들도 무작위 샘플링 기회가 열려 창의적인 결과를 낳습니다 \[37, 38\].
            
        *   $T \\rightarrow 0$ 일 때: 최상위 확률 단일 토큰만 부각되는 샤프닝(Sharpening)이 일어나 사실상 그리디 디코딩과 수리적으로 동일해집니다 \[37, 38\].
            
*   **탑-P(Top-P / Nucleus) 샘플링 영역 결정**: $$V^{(p)} = \\left{ w \\in \\mathcal{V} ;\\middle|; \\sum\_{j \\in \\text{sorted}} P(w\_j \\mid x\_{<t}) \\le p \\right}$$
    
    *   정렬된 토큰 중 누적 확률의 총합이 정량 임계치 $p \\in \[0, 1\]$ 범위 안에 들어오는 동적 어휘 후보군 $V^{(p)}$만 통과시키고 나머지는 점수를 $-\\infty$로 제거한 후 정규화하여 샘플링합니다 \[37, 38\].
        

### Key Tradeoffs / Procedures

*   **태스크별 최적 디코딩 하이퍼파라미터 권장 설정 가이드라인**:
    
    1.  **엄격한 팩트 기재 및 정밀 답변 (수학 수식 연산, 분류)**:
        
        *   _추천 세팅_: Greedy Decoding ($T=0.0$), Top-P와 Top-K 무효화 \[37, 38\].
            
        *   _트레이드오프_: 오독 가능성을 물리적으로 줄이고 일관된 결과를 출력하나, 창의성이 0으로 수렴하고 동일 문맥의 단기 반복 루프에 걸릴 위험이 상존합니다 \[37, 38\].
            
    2.  **구조화된 코드 및 API 생성, 기계 번역**:
        
        *   _추천 세팅_: Beam Search (Beam Width $k=3 \\sim 5$), 소량의 길이 페널티 적용 \[37, 38\].
            
        *   _트레이드오프_: 긴 문장 전반의 유기적 일관성을 확보하고 문법 파괴를 효과적으로 억제하지만, 기출되지 않은 어휘 사용율이 낮아져 대화형 챗봇에 적용 시 기계적이고 지루한 답변 패턴으로 일관하게 됩니다 \[37, 38\].
            
    3.  **창의적 시나리오 기획, 스토리 텔링, 아이디어 브레인스토밍**:
        
        *   _추천 세팅_: Stochastic Sampling ($T = 1.0 \\sim 1.2$, Top-P $p = 0.95$) \[37, 38\].
            
        *   _트레이드오프_: 독창적이고 표현의 가변 범위가 다채로운 결과물을 얻지만, 추론 길이가 길어질수록 극단적인 사실 왜곡(Hallucination) 및 논리적 자가당착에 빠질 확률이 정비례하여 증가합니다 \[37, 38\].
            

## 1.4 대조적 디코딩 (Contrastive Decoding, CD) 계열의 기술 분석

### Core Concept

*   \*\*대조적 디코딩(Contrastive Decoding)\*\*은 추가적인 학습 과정(Training-free) 없이 추론 단계에서 상호 보완적인 두 모델의 확률 밀도 간극을 이용해 생성 품질을 튜닝하는 기법입니다 \[108, 109\].
    
*   고성능 대형 모델을 \*\*에크스퍼트 모델(Expert, $P\_{\\text{exp}}$)\*\*로 동결하고, 의미 왜곡이나 자가 루프 퇴행을 조기에 빈번히 겪는 가벼운 소형 모델을 \*\*아마추어 모델(Amateur, $P\_{\\text{ama}}$)\*\*로 지정합니다 \[108, 110, 37\].
    
*   소형 모델이 자주 저지르는 공통의 문맥 왜곡 지표(반복적인 대명사, 평이한 조사 루프 등)를 대형 에크스퍼트의 예측 분포에서 빼주면, 대형 모델 내부에 숨어있던 정밀하고 의미적 정보 밀도가 높은 단어들이 표면 위로 강력하게 도출됩니다 \[109, 112\].
    

### Mathematical Formulation (If any)

*   **기본 대조적 디코딩 스코어 공식**: $$\\text{score}(w; c) = \\log P\_{\\text{exp}}(w \\mid c) - \\log P\_{\\text{ama}}(w \\mid c)$$
    
*   **에크스퍼트 신호 기반의 $\\alpha$-masking (Adaptive Plausibility Constraint) 제약 범위**: $$V(c) = \\left{ w \\in \\mathcal{V} ;\\middle|; P\_{\\text{exp}}(w \\mid c) \\ge \\alpha \\cdot \\max\_{w'} P\_{\\text{exp}}(w' \\mid c) \\right}$$
    
    *   $\\alpha$: 에크스퍼트가 부여한 최대 확률 점수 기준 임계비율(주로 0.1)입니다 \[111, 37\].
        
    *   이 제약 조건이 부재할 경우, 두 모델 모두 극단적으로 낮게 동조한 쓰레기 노이즈 단어가 대조 연산 수식의 뺄셈 효과로 인해 비정상적으로 높은 마진을 차지하여 최종 기출되는 치명적인 '명백한 맹목(Obvious Blindness)' 통계 왜곡을 일으키므로 반드시 마스킹 세트 $V(c)$ 내부에서만 argmax를 수행해야 합니다 \[113, 37\].
        
*   **Multi-Model Contrastive Decoding (MCD) 삼각 대조 외외삽 공식**: 단일 소형 모델과의 뺄셈에 의존할 경우 참값 토큰 자체의 신뢰도 우도까지 동반 훼손되어 최종 문장의 신뢰 점수가 떨어지는 단점을 개선하고자 제안되었습니다 \[12, 26\]. 의도적으로 편향과 왜곡을 강제한 \*\*Evil Model($P\_{\\theta^-}$)\*\*과 팩트 중심 정렬 데이터로 가중치를 굳힌 \*\*Truthful Model($P\_{\\theta^+}$)\*\*을 도입하여 삼각 간섭망을 수립합니다 \[12, 21\]. $$F\_{\\text{MCD}}(y\_t \\mid x, y\_{<t}) = \\log P\_{\\theta}(y\_t \\mid x, y\_{<t}) - \\alpha \\log P\_{\\theta^-}(y\_t \\mid x, y\_{<t}) + \\gamma \\log P\_{\\theta^+}(y\_t \\mid x, y\_{<t})$$
    
    *   $\\alpha, \\gamma$: 각각 Evil Model의 부정 왜곡 억제 페널티 계수 및 Truthful Model의 진실성 부각 가중치 계수입니다 \[18\].
        
*   **CCD (Confidence-Driven Contrastive Decoding)의 국소적 불안정성 탐지 공식**: 매 디코딩 추론 스텝마다 대조 연산을 강제하면 연산 오버헤드가 단독 생성 대비 최소 2배 이상 폭증합니다 \[116\]. 이에 따라 모델이 문맥상 아주 명확하게 예측을 처리하고 있는 신뢰 구간에서는 대조 필터를 비활성화하고, 정보 결핍이나 자가 반복으로 인해 로짓이 평탄화되는 순간만을 동적으로 측정하여 선별 제어합니다 \[31, 434\].
    
    *   각 토큰별 예측 신뢰도 $C\_i$: $$C\_i = -\\frac{1}{k} \\sum\_{j=1}^{k} \\log P\_i(j)$$
        
    *   슬라이딩 윈도우 버퍼 $\\mathcal{C}_{\\text{win}}^{(t)}$ 기준 하위 $q_{\\text{cd}}$-th 백분위수를 동적 교정 역치 $\\tau\_{\\text{cd}}$로 설정하여 다음의 지시 하이브리드 제어식을 구성합니다 \[431, 434\]: $$\\ell\_t^{\\text{final}} = \\begin{cases} (1+\\beta)\\ell\_t^{\\text{main}} - \\beta\\ell\_t^{\\text{cd}} & \\text{if } C\_t < \\tau\_{\\text{cd}} \\wedge t \\in \\mathcal{R} \\ \\ell\_t^{\\text{main}} & \\text{otherwise} \\end{cases}$$
        

### Key Tradeoffs / Procedures

*   **대조적 디코딩(CD)의 장단점 및 물리적 비용**:
    
    *   _장점_: 비싸고 불안정한 미세 조정 파인튜닝 학습 파이프라인을 완전히 대체하여 텍스트의 유창성(Fluency), 사실 일치성(Factuality), 추론 일관성을 오직 추론 레벨에서 직접 개선합니다 \[108, 121\].
        
    *   _단점_: 매 단계 에크스퍼트와 아마추어 모델 각각에 대해 순방향 추론(Forward pass)을 병렬 혹은 순차 가동해야 하므로 실시간 그래픽 메모리(VRAM) 점유율이 2배로 늘어나고 서비스 지연(Latency)이 동반 증가합니다 \[116, 37\].
        
    *   _대처 프로시저_: CCD(Confidence-Driven) 제어 루프를 적용하여 전체 토큰 시퀀스 중 15~20% 수준의 병목 구간에서만 대조 분기를 활성화하는 방식으로 물리 지연 비용을 효율화할 수 있습니다 \[426, 429\].
        

## 1.5 투기적 디코딩 (Speculative Decoding, SD) 기술 분석

### Core Concept

*   트랜스포머의 추론 연산은 매 단어를 생성할 때마다 수십 억 개의 가중치 매개변수를 그래픽 카드의 온보드 메모리(VRAM)에서 프로세서 코어로 업로드해야 하는 극단적인 \*\*메모리 대역폭 한계(Memory-Bandwidth Bound)\*\*에 직면해 있습니다 \[284, 344\].
    
*   \*\*투기적 디코딩(Speculative Decoding)\*\*은 병목 상태인 대형 타겟 검증 모델($M\_t$) 대신, 연산 및 파라미터 로딩 비용이 극도로 가벼운 소형 드래프트 모델($M\_d$)이 미래 토큰 $K$개 후보군을 순차적으로 초고속 사전 기출(Drafting / Speculation)하게 만듭니다 \[81, 280\].
    
*   이후 대형 타겟 모델은 $K$개의 후보 시퀀스를 입력 행렬로 구성하여 한 번의 대형 순방향 병렬 연산(Verification)을 수행하고, 이를 미리 계산해 둔 확률 분포와 대조하여 이론적으로 완벽한 일치(Lossless convergence)를 수리 유도함으로써 가속을 달성합니다 \[285, 286\].
    

### Mathematical Formulation (If any)

*   **무손실 통계적 리젝션 샘플링(Rejection Sampling) 합치 공식**: 타겟 모델의 누적 생성 정합성을 훼손하지 않기 위해, 제안된 임의의 드래프트 후보 단어 $x\_i$에 대해 다음과 같은 물리적 가부 검증(SpecJudge)과 동적 분포 정규 보정 연산(SpecNormalize)을 수행합니다 \[272, 346\].
    
    1.  **드래프트 토큰 직접 수용 조건 (SpecJudge)**: $$q\_d(x\_i \\mid x\_{:i-1}) \\le p\_t(x\_i \\mid x\_{:i-1}) \\quad \\text{인 경우 무조건 수락} \\quad$$ 그 외의 확률 전도 조건에서는 임의의 균등 분포 난수 $r \\sim \\text{Unif}(0, 1)$를 추출하여 다음 불등식 성립 시 통과시킵니다 \[272\]. $$r < \\frac{p\_t(x\_i \\mid x\_{:i-1})}{q\_d(x\_i \\mid x\_{:i-1})}$$
        
    2.  **불통과 시 동적 리샘플링 공식 (SpecNormalize)**: 특정 단계 $c$에서 드래프트 제안 토큰이 거절(Reject)당하면, 드래프트가 제안한 $c$ 이후의 모든 단어 후보군을 즉시 폐기(Discard) 처리합니다 \[281, 286\]. 이어서 타겟 모델이 연산 가동한 확률 값에서 드래프트의 예측값을 감산 및 클리핑하여 재조정한 다음 잔여 확률 밀도로부터 올바른 대체 토큰 $x\_c$를 재추출합니다 \[272, 346\]. $$x\_c \\sim \\text{normalize}\\left(\\max\\left(0, ; p\_t(x \\mid x\_{:c-1}) - q\_d(x \\mid x\_{:c-1})\\right)\\right)$$
        

### Key Tradeoffs / Procedures

*   **투기적 디코딩의 병렬 검증 수용 알고리즘 절차**:
    
    1.  **Drafting**: 드래프트 모델이 현재 컨텍스트를 기반으로 타겟 윈도우 크기 $K$개만큼의 미래 토큰 후보군 $x\_1, \\dots, x\_K$를 생성합니다 \[281\].
        
    2.  **Verification**: 타겟 모델에 수집된 $K$개 토큰 구조를 다차원 행렬로 배치하여 $K+1$개의 예측 분포를 단 1회의 병렬 Forward Pass 연산으로 계산해냅니다 \[285\].
        
    3.  **Acceptance Loop**: $i=1$부터 $K$까지 검증 수식을 차례대로 돌려 통과(Accept) 여부를 점검합니다 \[346\].
        
    4.  **Bifurcation Point**: 최초 불합격 시점 $c$가 발생하면 $c+1$ 이후의 모든 잔여 후보군은 완전 삭제하며, $c$ 시점은 보정 분포 $p'(x)$에서 재생성한 올바른 토큰으로 갈아 끼운 뒤 다음 라운드를 시작합니다 \[286, 346\].
        
    
    *   _주요 트레이드오프_: 드래프트 모델의 매개변수가 작을수록 후보를 작성하는 속도는 빨라지지만 타겟 모델의 예측 분포와의 수학적 괴리가 심해져 수용률($\\alpha$)이 급감하여 실질적인 가속 비율이 줄어듭니다 \[37, 573\]. 반대로 드래프트 모델이 너무 무거우면 개별 Drafting 지연시간 자체가 증가해 병렬 가속 효과를 무력화시킵니다 \[37, 573\].

## Chapter 2. RAG 및 컨텍스트 제어 기술 (Retrieval Augmentation)

## 2.1 REPLUG 및 LSR 리트리버 정렬 방법론

### Core Concept

*   **REPLUG**는 기존 트랜스포머 아키텍처가 지닌 긴 컨텍스트 윈도우 수용 시의 $O(N^2)$ 자가주의 계산 효율 붕괴 현상을 막기 위해 설계된 블랙박스형 지식 주입 프레임워크입니다 \[575\].
    
*   검색 도구(Retriever)가 획득한 고차원 상위 $K$개의 관련 도큐먼트들을 질문 컨텍스트 앞부분에 병렬적으로 결합(Input Reformulation)하여 각각 가볍게 통과시킨 뒤, 최종 출력 확률 레벨에서 앙상블 우도로 종합 처리합니다 \[575\].
    
*   **LSR (LM-Supervised Retrieval)** 기술은 주입되는 정보 도큐먼트 간의 상호작용 우도를 검색기가 더 정밀하게 예측하도록, 검색 벡터 인코더 가중치를 언어 모델의 다음 토큰 생성 우도 신호를 교사(Supervised) 삼아 역전파 정렬하는 동적 튜닝 모델입니다 \[22\].
    

### Mathematical Formulation (If any)

*   **REPLUG 병렬 앙상블 출력 통합 공식**: $$p(y \\mid x, \\mathcal{D}') = \\sum\_{d \\in \\mathcal{D}'} p(y \\mid d \\circ x) \\cdot \\lambda(d, x)$$
    
    *   $\\mathcal{D}'$: 질문에 대응해 수집된 $K$개의 외부 참조 도큐먼트 개별 집합입니다 \[575\].
        
    *   $d \\circ x$: 도큐먼트 $d$와 질문 $x$를 1차원으로 직렬 결합한 입력 문자열입니다 \[575\].
        
    *   $\\lambda(d, x)$: 질문 벡터와 도큐먼트 조각 간의 밀집 코사인 벡터 공간 유사도로 규정하는 가중치 앙상블 계수입니다 \[575\]. $$\\lambda(d, x) = \\frac{\\exp(s(d, x))}{\\sum\_{d' \\in \\mathcal{D}'} \\exp(s(d', x))}$$
        
*   **LSR 최적화를 위한 Kullback-Leibler 발산 손실 함수**: 검색기의 예측 확률 $P\_R(d \\mid x)$를 정방향 타겟 값으로 정렬하기 위해, 언어 모델이 문서 주입 후 실제 정답 토큰을 수식 복원할 때 계산하는 동적 분포 $Q\_{\\text{LM}}(d \\mid x, y)$와의 정보 거리를 최소화하도록 역전파 손실 $\\mathcal{L}$를 최적화합니다 \[22\]. $$P\_R(d \\mid x) = \\frac{\\exp(s(d,x) / \\gamma)}{\\sum\_{d' \\in \\mathcal{D}'} \\exp(s(d',x) / \\gamma)}$$ $$Q\_{\\text{LM}}(d \\mid x, y) = \\frac{\\exp(P\_{\\text{LM}}(y \\mid d, x) / \\beta)}{\\sum\_{d' \\in \\mathcal{D}'} \\exp(P\_{\\text{LM}}(y \\mid d', x) / \\beta)}$$ $$\\mathcal{L} = \\frac{1}{|\\mathcal{B}|} \\sum\_{x \\in \\mathcal{B}} \\text{KL}\\left(P\_R(d \\mid x) ;\\parallel; Q\_{\\text{LM}}(d \\mid x, y)\\right)$$
    
    *   $\\gamma, \\beta$: 각 분포 밀도의 날카로움을 보정하는 제어 하이퍼파라미터 스케일러입니다 \[22\].
        
    *   $\\mathcal{B}$: 학습용 입력 질문 배치 단위 세트입니다 \[22\].
        

### Key Tradeoffs / Procedures

*   **REPLUG LSR 정렬을 위한 엔드투엔드 처리 루틴**:
    
    1.  **Forward Document Retrieval**: 검색기가 입력 질문 $x$를 벡터 임베딩한 후, 말뭉치 인프라에서 $K$개의 관련 도큐먼트 $d\_1, \\dots, d\_K$를 획득합니다 \[575\].
        
    2.  **Independent Next Token Computation**: 각각의 조각 문서 결합 컨텍스트 $\[d\_i \\circ x\]$에 대해 모델의 순방향 연산을 태우고 타겟 출력 $y$에 대한 예측 분포를 생성합니다 \[575\].
        
    3.  **Construct LM-supervised Signal**: 위 식의 $Q\_{\\text{LM}}$에 맞춰 실제 정답 단어 복원에 기여한 문서가 더 가중되도록 우도 타겟 분포를 수립합니다 \[22\].
        
    4.  **Gradient Update**: 검색 결합 가중치 $P\_R(d \\mid x)$의 KL-Divergence 차이를 편미분 계산하여 임베딩 인코더 신경망 파라미터를 하강 업데이트합니다 \[22\].
        
    
    *   _장단점_: 기존 고정 RAG 시스템의 한계(질문과 완전히 겹치는 조사가 많다고 무익한 문서를 최상위로 추천하여 생성 완성도를 무너뜨리는 현상)를 LM의 실제 생성 효율 기반으로 직접 정렬함으로써 검색 인프라 성능을 대폭 끌어올립니다 \[22\]. 다만 학습 단계에서 각 질문 배치마다 언어 모델 순방향 계산을 모든 검색 수집 도큐먼트 가닥별로 개별 연산해야 하므로 훈련 비용이 큽니다 \[22\].

## Chapter 3. 대규모 언어 모델 평가 방법론 (Evaluation of LLMs)

## 3.1 정답(Ground-Truth) 기반 평가 방법론

### Core Concept

*   **정답 기반 평가**는 데이터셋 자체에 모델이 복원 기출해야 할 황금 가이드 정답(Golden Reference)이 텍스트 문자열 형태로 명시되어 있는 환경에서 구동됩니다 \[579, 580\].
    
*   주로 MMLU와 같은 4지선다형 객관식, Natural Questions(NQ)와 같은 단답형 질의응답, XSum과 같은 극단적 레퍼런스 의미 요약 태스크를 통해 모델 성능의 기본 구조와 팩트 정밀도를 평가합니다 \[582, 583, 585\].
    

### Mathematical Formulation (If any)

*   **EM (Exact Match) 스코어의 지시 함수**: $$\\text{EM}(Y\_{\\text{pred}}, Y\_{\\text{ref}}) = \\begin{cases} 1 & \\text{if } \\text{normalize}(Y\_{\\text{pred}}) = \\text{normalize}(Y\_{\\text{ref}}) \\ 0 & \\text{otherwise} \\end{cases}$$
    
    *   대소문자 제거, 기호 제거, 불필요 관사 교정 등의 전처리 정규화 과정을 거친 예측 출력값 $Y\_{\\text{pred}}$와 참조 정답 문자열 $Y\_{\\text{ref}}$가 글자 수준에서 한 글자도 틀리지 않고 완전히 일치할 때만 1점을 획득합니다 \[584\].
        
*   **F1 Score의 정밀도(Precision) 및 재현율(Recall) 조화평균**: $$\\text{Precision} = \\frac{|P \\cap A|}{|P|}$$ $$\\text{Recall} = \\frac{|P \\cap A|}{|A|}$$ $$F\_1 = \\frac{2 \\times \\text{Precision} \\times \\text{Recall}}{\\text{Precision} + \\text{Recall}} = \\frac{2}{\\text{Recall}^{-1} + \\text{Precision}^{-1}}$$
    
    *   $P$: 예측 텍스트 내 형태소/단어 집합이며, $A$: 정답 레퍼런스 내 형태소/단어 집합입니다 \[584\].
        
    *   단어 순서와 무관하게 팩트 유무 및 어휘 유실 정도를 보완 추적할 수 있어 단답형 생성 태스크의 표준 매칭 지표로 널리 통용됩니다 \[584\].
        

### Key Tradeoffs / Procedures

*   **형태소 단위 정규화 및 NQ F1 연산 파이프라인**:
    
    1.  모델이 기출한 최종 생성 원시 문자열에서 문장 부호(Punctuation) 및 구두점을 정규식으로 삭제 처리합니다 \[584\].
        
    2.  무의미한 관사(English의 경우 `a`, `an`, `the` 등)를 문자열 정규화 단계에서 필터링 소거합니다 \[584\].
        
    3.  화이트 스페이스 공백 기준으로 분할된 단어 목록을 각각 예측 세트 $P$와 골드 정답 세트 $A$로 전환합니다 \[584\].
        
    4.  교집합 $P \\cap A$ 세그먼트의 수를 산출하여 Precision, Recall을 전개한 뒤 최종 조화 평균 F1 스코어를 산출합니다 \[584\].
        
    
    *   _수리적 트레이드오프_: EM 스코어는 극도로 보수적인 평가 척도로, 의미가 완벽히 동일하더라도 표기방식의 근소한 차이(예: 'Barack Obama' vs 'Obama')가 발생하면 바로 0점 처리합니다 \[584\]. F1 스코어는 이 유연성 한계를 완화시켜 주지만 단어의 배열 순서나 반어법, 부정어 유무 등의 고차원 문맥적 의미 차이를 완전히 놓쳐버리는 통계적 맹점을 지니고 있습니다 \[584\].
        

## 3.2 정답(Ground-Truth)이 불분명한 개방형 요약 및 생성 평가 방법론

### Core Concept

*   **개방형 텍스트 생성** 태스크는 정해진 하나의 유일 정답이 존재하지 않으므로, 사실 일치도, 흐름의 일관성, 문장의 자연스러움 등을 평가하기 위한 대안 척도가 수반되어야 합니다 \[586, 587\].
    
*   모델 내부 혹은 외부 검증 모델을 이용한 통계적 혼란도 분석 기법과 함께, 최근에는 프론트엔드 최정상급 모델을 자율 평가 도구로 승격시키는 기술들이 적극 활용되는 추세입니다 \[588, 37\].
    

### Mathematical Formulation (If any)

*   **혼란도(Perplexity, PPL) 지표 공식**: 문장 내부의 각 토큰 위치에서 모델 스스로가 예측 확률로 부여하는 조건부 우도들의 기하평균의 역수로 정의됩니다 \[588\]. $$\\text{Perplexity} = \\sqrt\[N\]{\\frac{1}{p(w\_1, w\_2, \\dots, w\_N)}} = \\sqrt\[N\]{\\frac{1}{\\prod\_{i=1}^{N} P(w\_i \\mid w\_{<i})}} = \\exp\\left(-\\frac{1}{N} \\sum\_{i=1}^{N} \\log P(w\_i \\mid w\_{<i})\\right)$$
    
    *   $N$: 전체 평가 시퀀스 토큰 총수입니다 \[588\].
        
    *   $P(w\_i \\mid w\_{<i})$: 모델이 이전 맥락을 바탕으로 $w\_i$를 예측할 확률입니다 \[588\].
        
    *   수치 지표가 하락하여 낮은 값을 보장할수록 모델이 테스트 데이터셋을 막힘없이 매끄럽고 일관되게 복원 구축해 나가고 있음을 대변합니다 \[588\].
        

### Key Tradeoffs / Procedures

*   **G-Eval 및 LLM-as-judge 정성적 평가 자동화 워크플로우**:
    
    1.  **Evaluation Criteria Definition**: 프롬프트 상단에 평가하고자 하는 성격(예: 문장의 일관성, 논리 전개도 등)의 평가 기준과 1~5점 척도 가이드라인을 상세 정의 주입합니다.
        
    2.  **Contextual Evaluation**: 생성된 시퀀스 결과물과 최초 입력을 평가 모델에 함께 전달합니다.
        
    3.  **Token Likelihood Weighing**: G-Eval에서는 단순 텍스트 기출 점수 획득 오류를 예방하기 위해, 평가 모델이 점수 토큰(예: "1", "2", "3", "4", "5")들을 각 위치에서 기출할 때 계산되는 Softmax 우도값을 기반으로 가중 기하평균 평점을 재계산하여 바이어스를 제거합니다.
        
    
    *   _장단점_: 전통적인 통계 기반 지표(Perplexity 등)가 인간이 인지하는 실제 시맨틱 품질이나 생성의 사실성과 일치하지 못하는 고질적인 오독 현상을 인간 주관 평가단 수준에 준하도록 완벽히 해소해 줍니다. 그러나 평가 도구로 기동되는 판단 주체 모델(GPT-4 등)이 지닌 고유의 편향(자체 생성 단어를 과도하게 선호하는 Self-preference, 먼저 제시된 답변에 높은 점수를 남발하는 Position bias 등)에 취약하며 API 가동 비용 부담이 증가합니다.
