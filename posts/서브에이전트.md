Claude Code에서 서브에이전트를 쓰는 방법은 크게 세 가지예요.

**① 그냥 말로 시키기 (제일 쉬움)**

Claude Code한테 이렇게 부탁하면 알아서 서브에이전트를 띄워요:

```text
서브에이전트를 써서 이 프로젝트에서 로그인 관련 코드가 어디 있는지 찾아줘.
파일 내용은 다 가져오지 말고, 어느 파일에 뭐가 있는지 요약만 정리해줘.
```

"서브에이전트를 써서" 또는 "탐색 에이전트로" 라고 붙이면 별도 창에서 뒤지고 **요약만** 돌려줍니다.

**② 전용 서브에이전트를 미리 만들어두기**

자주 쓰면 `.claude/agents/` 폴더에 파일로 정의해둬요. 그럼 매번 설명 안 해도 돼요.

`.claude/agents/explorer.md`:

```markdown
---
name: explorer
description: 코드베이스에서 특정 기능이 어디 있는지 찾는 탐색 전용 에이전트.
             파일 전체가 아니라 위치와 요약만 반환한다.
tools: Read, Grep, Glob
---

너는 코드 탐색 전문가다. 요청받은 주제와 관련된 파일을 찾아
"어느 파일 / 무슨 역할 / 핵심 라인" 형식으로 요약만 보고해라.
파일 전체 내용은 절대 그대로 붙이지 마라.
```

이렇게 해두면:

```text
explorer 에이전트로 결제 로직 찾아줘
```

`tools: Read, Grep, Glob` 처럼 **읽기 도구만** 쥐여주는 게 포인트예요 — 탐색만 하고 실수로 코드를 수정 못 하게.

**③ 병렬로 여러 개 (고급)**

큰 작업이면 여러 서브에이전트를 동시에:

```text
서브에이전트 3개를 병렬로 써서:
1번은 인증, 2번은 결제, 3번은 알림 관련 코드를 각각 찾아서 요약해줘.
```

* * *

**핵심 팁 하나**: 서브에이전트한테 시킬 때 **"요약만, 전체 붙이지 마"** 를 꼭 명시하세요. 안 그러면 서브에이전트가 파일 내용을 통째로 돌려줘서 컨텍스트 절약 효과가 사라집니다.

**언제 쓰냐 감**: "코드 다 뒤져서 찾아줘", "로그 100개 분석해줘", "이 라이브러리 전체 구조 파악해줘" 처럼 **읽을 건 많은데 결론은 짧은** 작업 → 서브에이전트. 반대로 파일 한두 개 고치는 작업은 그냥 메인이 하는 게 나아요.
