**MCP = Model Context Protocol (모델 컨텍스트 프로토콜)** — AI(모델)에게 외부 도구·데이터를 붙이는 **표준 규격**이에요.

**한 줄 비유**: **AI의 USB-C 포트**라고 보면 돼요. USB-C 하나로 모니터·키보드·충전기 다 꽂듯이, MCP라는 표준 하나로 AI에 GitHub·데이터베이스·Slack·구글드라이브 같은 걸 다 꽂을 수 있어요.

**왜 필요하냐**

AI 자체는 **자기 머릿속 지식 + 지금 대화창**만 알아요. 내 GitHub에 뭐가 있는지, 우리 회사 DB에 무슨 데이터가 있는지는 몰라요. 그걸 연결해주는 통로가 MCP예요.

*   MCP 없으면 → "내 DB에서 어제 가입자 몇 명이야?" 물어도 AI가 모름
    
*   MCP로 DB를 연결하면 → AI가 **직접 DB를 조회해서** 답해줌
    

**구조 (간단히)**

```
AI (Claude 등)  ←── MCP ──→  MCP 서버 (GitHub / DB / Slack ...)
```

*   **MCP 서버** = 특정 도구를 AI가 쓸 수 있게 감싸놓은 프로그램. GitHub용, DB용 등 종류별로 있어요.
    
*   AI는 MCP 서버가 제공하는 **도구 목록**을 받아서, 필요할 때 그걸 호출해요.
    

**왜 "표준"이 중요하냐**

예전엔 AI마다, 도구마다 연결 방식이 제각각이라 매번 새로 만들어야 했어요. MCP는 **한 번 만든 MCP 서버를 여러 AI에서 재사용**할 수 있게 통일한 거예요. (Anthropic이 2024년에 공개했고 지금은 업계 표준처럼 쓰여요.)

* * *

**정리**: MCP = AI에 외부 세계(도구·데이터)를 꽂는 **표준 커넥터**. 하네스 관점에선 **AI의 감각·행동 범위를 넓히는 확장 슬롯**이에요.

참고로 지금 이 대화에도 여러 MCP가 붙어 있어요 — 아까 스터디 자료 만들 때 쓴 것들 중에 Slack·Notion·GitHub 같은 커넥터가 다 MCP예요.
