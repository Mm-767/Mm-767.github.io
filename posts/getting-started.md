This file exists to keep `/posts` non-empty. Astro's content-collection glob
loader only watches this directory for new files while `astro dev` is
running if it isn't empty at server start — with zero files here, posts
saved via the editor's "Save Post" button won't show up on the home page
until the dev server restarts. `draft: true` keeps this out of every public
listing and page.

Don't delete this file unless you're okay restarting `astro dev` after
adding your first real post.
